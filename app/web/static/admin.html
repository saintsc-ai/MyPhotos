새 폴더 추가

체크 시 인덱싱/EXIF/썸네일은 정상 동작하지만 다음은 거부됩니다: 라이트박스 🗑 삭제, 다중 선택 일괄 삭제, 중복 popover "이것 빼고 휴지통", 관리 → 중복 제거 자동정리 / 그룹 정리.
잠깐 정리할 일이 있으면 위 사진 폴더 목록의 RO/RW 배지를 클릭해서 즉시 토글할 수 있습니다 (root를 다시 등록할 필요 없음).
경로 입력 안내
  • 도커: docker-compose.yml에서 마운트한 컨테이너 안 경로를 입력하세요 (기본 /photos, 추가 마운트면 /photos2 등). 호스트 경로(/volume1/photo)는 컨테이너에서 보이지 않습니다.
  • 시놀로지 직접 설치: 호스트 절대 경로 그대로 (/volume1/photo).
  • 상태가 접근 불가로 뜨면 폴더 권한 문제입니다 — Synology Photos가 만든 폴더는 보통 d---------+(ACL 전용)이라 컨테이너 UID로는 못 읽습니다. 호스트에서 sudo chmod 777 /volume1/photo 한 번이면 해결됩니다.

추가 후 “시험”(200장)으로 한 번 돌려보고, 문제가 없으면 limit을 비우고 “스캔”으로 풀스캔하세요.

Live Photo 짝 매칭

iPhone Live Photo는 같은 이름의 .HEIC + .MOV 두 파일로 저장됩니다. 같은 폴더·같은 stem·같은 날짜의 image↔video 짝을 찾아 양방향 연결합니다. 연결 후에는 타임라인에 사진(HEIC)만 표시되고, 라이트박스의 ▶ Live 토글로 동영상을 재생할 수 있습니다.

※ 풀스캔 후 새로 들어온 짝들도 이 버튼으로 한 번에 정리 가능 (idempotent)

ML 자동 분류 (YOLO + CLIP + 얼굴)

썸네일이 있는 사진을 ML 워커가 분석합니다 — YOLO는 사람·동물·물체, CLIP은 풍경/음식/셀카 같은 분위기, 얼굴 모듈은 같은 사람의 사진을 묶어 클러스터를 만듭니다. 결과는 갤러리 → 주제 탭과 검색바 태그 필터에 반영됩니다.

휴지통

삭제한 사진의 원본 파일은 data/trash/<id>/에 보관됩니다. 여기서 선택해서 원래 위치로 복구하거나 영구 삭제할 수 있습니다. 영구 삭제는 파일과 DB 행, 관련된 코멘트/태그/별점까지 모두 지우므로 되돌릴 수 없습니다.

중복 사진

같은 파일 내용(SHA-256 일치)이 여러 폴더에 인덱싱된 경우입니다. 썸네일은 이미 자동으로 공유되니 정리는 카탈로그 정돈 목적이고, 버튼을 누르면 그 그룹에서 유지로 선택된 항목 외 나머지를 휴지통(data/trash/)으로 이동합니다. 원본 파일은 휴지통에서 복구 가능합니다.

유지 선택 기준 — 각 그룹의 가장 오래된 1장을 기본 유지로 제안합니다: ① 촬영시각(EXIF) → ② 파일 mtime → ③ 폴더/경로 알파벳 순서로 가장 앞선 항목. 다른 사진을 유지하고 싶다면 카드 우상단의 유지하기 핀을 누르세요. ★ 유지 보존 → 휴지통 정리 대상 🔒 RO 읽기전용 폴더 (정리 불가)

대상 DSN 검증 / 마이그레이션 dry-run

새 백엔드로 옮기기 전에 ① 연결 가능 여부, ② 마이그레이션이 얼마나 큰지(원본/대상 행 수)를 미리 봅니다. 실제 데이터 복사는 아래 SSH 절차로 수행하세요 — 라이브 트래픽 중에 DB를 바꾸면 잡 큐 상태가 어긋날 수 있어 의도적으로 자동화하지 않습니다.

전환 절차 (SSH)

  1. 위 백업 버튼으로 현재 DB 스냅샷을 확보합니다.
  2. 위 dry-run으로 대상 DSN과 행 수가 의도와 맞는지 확인합니다.
  3. NAS에서 모든 서비스를 멈춥니다: 
    sudo systemctl stop myphotos-api myphotos-worker myphotos-ml-worker myphotos-watcher
  4. 마이그레이션 실행 (양방향 가능): 
    .venv/bin/python scripts/migrate-db.py SRC_URL DST_URL --drop
  5. config/local.toml 의 [database] url 을 새 DSN으로 변경.
  6. 서비스 재시작: 
    sudo systemctl start myphotos-api myphotos-worker myphotos-ml-worker myphotos-watcher
  7. 이 페이지를 새로고침해서 위 "데이터베이스" 정보가 새 백엔드로 표시되는지 확인.
